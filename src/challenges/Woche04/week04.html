Woche 04: CSS Component Library + Kontaktseite

Beschreibung

Diese Woche geht es um wiederverwendbare Bausteine in HTML & CSS. Du erstellst deine eigene kleine CSS Component Library, bestehend aus Buttons, Cards, Textfeldern, Checkboxen, Tags, Überschriften und Fließtext. Alle Komponenten werden in einer zentralen CSS-Datei (components.css) definiert – mit klaren, eindeutigen kebab-case Klassennamen.
Anschließend setzt du diese Komponenten auf einer separaten, responsiven Kontaktseite ein. Die Kontaktseite soll mindestens Desktop- und Mobile-Ansicht unterstützen und zeigt, dass deine Komponenten flexibel wiederverwendbar sind.

Du arbeitest ohne JavaScript und ohne Frameworks – nur HTML und CSS. Achte außerdem auf Hover-, Focus- und Active-Zustände, sowie kleine Transitionen für ein modernes UI-Gefühl.

Abgabe: am MOntag, 01. Dezember 2025.

Ziel

Nutze HTML und CSS, um:

  • eine saubere und wiederverwendbare Component Library aufzubauen
  • verständliche, eindeutige CSS-Klassen im kebab-case zu erstellen
  • ein responsives Layout für die Kontaktseite umzusetzen
  • moderne UI-Elemente mit Hover/Fokus/Pressed-Effekten zu gestalten
  • deine Komponenten sinnvoll zu strukturieren und zu kommentieren
  • ein kleines Mini-Design-System für dein Projekt zu entwickeln

Anforderungen

  • Verwende eine zentrale CSS-Datei (components.css)
  • Erstelle zunächst eine Seite, die deine Komponenten zeigt (z. B. components.html)
  • Setze folgende Komponenten um:
    • Buttons (mind. Primary + Secondary)
    • Input-Textfeld
    • Checkbox
    • Tags (Labels/Chips)
    • Card
    • Überschriften & Fließtext
    • Optional: weitere UI-Elemente wie Alerts, Badges oder weitere Varianten
  • Nutze diese Komponenten anschließend auf einer separaten Kontaktseite (contact.html):
    • Die Seite soll responsiv sein
    • Pflicht: Einsatz von Überschriften, Text, Input, Checkbox, Button
    • Optional: Tags oder Cards zur Strukturierung
    • Beispiel-Aufbau: Eine Card mit dem gesamten Kontaktformular
  • Komponenten sollen ausschließlich über CSS-Klassen funktionieren
    • keine Inline-Styles
    • kein JavaScript
  • Verwende Kommentare in deiner CSS-Datei (z. B. /* buttons */, /* inputs */), um Struktur zu schaffen
  • Achte auf Animationen und Zustände:
    • Hover
    • Focus
    • Pressed/Active
    • kleine Transitionen
  • Arbeite klar strukturiert und modern – orientiere dich gern an echten UI-Libraries wie Material, Bootstrap oder Tailwind (nur Inspiration, nicht kopieren)
