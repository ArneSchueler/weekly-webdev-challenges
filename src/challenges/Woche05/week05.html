Woche 05: JS-Stopuhr mit Runden

Beschreibung

Diese Woche geht es um deinen ersten interaktiven JavaScript-Use Case im Browser: eine Stopuhr. Du baust eine kleine Web-App, die Zeit ab 0 in Millisekunden hochzählt und im Interface anzeigt. Die Stopuhr soll per Buttons gesteuert werden: Start, Stop, Clear und Runde.
Über Runden (Lap-Funktion) speicherst du Zeitpunkte, die in einer Liste unter der Stopuhr angezeigt werden.

Du nutzt die Inhalte aus JS 1: Variablen, Funktionen, Events, DOM-Manipulation und einfache CSS-Manipulation über JavaScript. Frameworks sind nicht erlaubt – nur pures HTML, CSS und JavaScript.

Abgabe: am Montag, 08. Dezember 2025.

Ziel

Nutze HTML, CSS und JavaScript, um:

  • eine funktionierende Stopuhr mit Start/Stop/Clear/Runden umzusetzen
  • JavaScript-Funktionen für Logik und Zeitberechnung zu schreiben
  • Events (click) zu nutzen, um Buttons mit Funktionalität zu verbinden
  • DOM-Elemente gezielt auszuwählen und ihren Inhalt zu verändern
  • mit CSS-Klassen Zustände wie „läuft“ oder „gestoppt“ sichtbar zu machen
  • saubere Struktur in HTML, CSS und JS beizubehalten

Anforderungen

  • Lege mindestens diese Dateien an:
    • index.html
    • style.css
    • script.js
  • Erstelle das UI der Stopuhr:
    • ein Anzeigefeld für die Zeit (z. B. mm:ss.mmm)
    • Buttons: Start, Stop, Clear/Reset, Runde
    • einen Bereich / eine Liste für Rundenzeiten
  • Funktionalität der Stopuhr:
    • Zeit startet bei 0 ms und zählt nach Klick auf Start hoch
    • Stop pausiert die Zeit, erneutes Start läuft von dort weiter
    • Clear setzt Zeit auf 0 und löscht alle Runden
    • Runde speichert die aktuelle Zeit und zeigt sie als Eintrag (z. B. „Runde 1 – 00:12.350“)
    • Zeitbasis intern in Millisekunden, Darstellung sauber formatiert
  • JavaScript-Vorgaben:
    • nutze eigene Funktionen (z. B. für Start, Stop, Clear, Runden, Formatierung)
    • Events über addEventListener("click", ...), kein onclick im HTML
    • DOM-Auswahl mit document.getElementById oder querySelector
  • CSS & UX:
    • Buttons sollen klar erkennbar und gut klickbar sein (auch auf Mobile)
    • Zeige mit CSS, ob die Stopuhr gerade läuft oder gestoppt ist (z. B. Button-Zustände)
    • nutze einfache Hover- und Focus-Zustände für bessere Bedienbarkeit
  • Optional (Bonus):
    • Unterschied zwischen Gesamtzeit und letzter Rundenzeit anzeigen
    • Keyboard-Shortcuts (z. B. Space = Start/Stop, R = Runde, C = Clear)
    • eine leichte Animation beim Start/Stop des Timers
